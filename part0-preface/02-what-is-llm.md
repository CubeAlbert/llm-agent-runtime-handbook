# LLM到底是什么

## 一句话定义：给定上文，预测下一个最可能的 token

如果你刚读完第一章，可能会觉得这节有些眼熟——上一节我们在讨论"为什么要学习 LLM"时已经提到了 LLM 的本质，这一节我们来更详细地拆开讲。

在此之前，我们要先理解两个概念，一个叫 token，一个叫上文。

Token 是什么？Token 可以认为是一段文字被切分后的最小单元，可以是一个字、一个词，也可以是半个词。例如，"Hello world" 会被切成 `["Hello", "world"]` 两个 token；"我喜欢吃苹果" 可能被切成 `["我", "喜欢", "吃", "苹果"]` 四个 token。

那上文又是什么？上文就是模型当前看到的所有文字，包括三部分：① 你之前的提问和模型之前的回答（也就是对话历史）；② 你这一次的输入；③ 模型自己已经生成、还没有结束的那部分内容。这些都属于上文。

搞清楚这两件事之后，LLM 在做的事情就清楚了：给定这些上文，模型按概率挑出一个 token 放到生成内容的末尾，再把新生成的 token 也加入上文，再挑下一个 token，如此循环，直到输出结束。

至于模型是怎么训练出来的、概率具体怎么算的、Transformer 内部又长什么样，这些会在本书后面的章节（训练、Transformer、采样）逐步展开。这一节我们只关心"它在做什么"，不关心"它怎么做到的"。

你可能已经发现了——LLM 并没有在回答你"是什么"，而是在告诉你"接下来我最可能写什么"。看起来很像，做的事情完全不同。这两种回答的差别，我们留到下一节，从概率的角度继续拆解。
